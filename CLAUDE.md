# yokosuka-todo

横須賀市の構造的な課題を、市民が読める形で網羅的に整理・公開する静的サイト。
雛形: https://fladdict.github.io/japan-todo (構成思想を参考。コードは独自)

## ベンチマーク(全ページ共通の判断軸)

**「持続可能性をもった横須賀市の発展と、市民のQOL(生活の質)の両立」**

すべての課題・施策は「これは持続可能性に効くか? 市民のQOLに効くか? 両者が衝突するならどう折り合うか?」で評価する。

## リポジトリ構成

```
yokosuka-todo/
├── CLAUDE.md            ← このファイル(運用ルール)
├── README.md
├── STYLE_GUIDE.md       ← 文章ルール(必読)
├── research/            ← リサーチ生データ(テーマ別Markdown、出典URL付き)※非公開扱いの作業領域
├── templates/
│   └── issue.html       ← 課題ページの雛形(コピーして使う)
└── docs/                ← 公開されるサイト本体(GitHub Pages: main / docs)
    ├── index.html       ← トップ(カテゴリ別の課題カード一覧)
    ├── about.html       ← サイトの目的・つくり方・免責
    ├── sources.html     ← 出典・データ集
    ├── assets/style.css
    └── issues/
        └── <category>/<slug>.html
```

## 課題ページの必須セクション(順番も固定)

1. **一言でいうと** — 3行以内+メタファ(例え話)1つ
2. **いま何が起きている?** — 数値データ。必ず出典つき
3. **なぜこうなった?** — 原因の構造を箇条書きで
4. **放っておくとどうなる?** — 5年後・10年後の時間軸で
5. **守りの一手** — 暮らしを支える・悪化を止める施策
6. **攻めの一手** — 伸ばす・新しくする施策
7. **どう進める?** — 短期(1〜2年)/中期(3〜5年)/長期(10年)
8. **トレードオフ — 全部はできない** — 何かを選べば何かをあきらめる、を明示
9. **オススメの構成** — このサイトとしての推奨パッケージ(守り×攻めの組み合わせ)
10. **どう実現する?** — お金(財源)・人(体制)・合意づくり
11. **もっと知りたい人へ** — 専門的な内容は外部リンクへ誘導
12. **出典** / **更新履歴**

メタ情報(ページ上部): 緊急度●1〜5 / 深刻度●1〜5 / 時間軸(今すぐ・5年以内・10年以内) / タグ / 最終確認日

## 更新パイプライン(この順で行う)

1. **情報収集** — WebSearch/WebFetchで一次資料(市の統計・計画・国の統計)を優先して収集。`research/`にテーマ別Markdownで保存(事実+出典URL+信頼度)
2. **分析** — ベンチマーク(持続可能性×QOL)で守り/攻め/トレードオフに構造化
3. **静的ページの更新** — `templates/issue.html`をコピーして`docs/issues/<category>/<slug>.html`を作成・更新。STYLE_GUIDE.md準拠
4. **ファクトチェック** — ページ内の全数値を`research/`の出典と突き合わせる。出典で確認できない数値は削除するか「推定」と明記
5. **インデックス更新** — `docs/index.html`のカード一覧、`docs/sources.html`、ページ内の最終確認日を更新
6. **アップロード** — `git add -A && git commit && git push`。GitHub Pagesが自動反映

## 技術ルール

- ビルドツール・フレームワーク・JSフレームワークは使わない。素のHTML+CSS(+最小限のJS)
- CSSは`docs/assets/style.css`の1ファイルのみ
- リンクはすべて相対パス(プロジェクトPages `/yokosuka-todo/` 配下で動くように)
- 課題ページから戻るリンク: `../../index.html`
- 文字コードUTF-8、`lang="ja"`、viewport必須
